# 开发路线

本项目按阶段推进。每个阶段完成后必须总结本阶段内容、关键文件、运行或测试方式，并尽量运行 `flutter analyze`。

## 阶段列表

- 第 0 阶段：环境检查与项目规则文档。
- 第 1 阶段：初始化 Flutter Windows 项目。
- 第 2 阶段：建立目录结构和基础依赖。
- 第 3 阶段：建立 UI 设计系统。
- 第 4 阶段：建立通用 UI 组件。
- 第 5 阶段：接入 SQLite + Drift。
- 第 6 阶段：实现数据库核心表。
- 第 7 阶段：实现 Repository 层。
- 第 8 阶段：实现 Service 层基础能力。
- 第 9 阶段：订单模块。
- 第 10 阶段：客户模块。
- 第 11 阶段：产品模块。
- 第 12 阶段：厂家模块。
- 第 13 阶段：发货模块。
- 第 14 阶段：费用和利润模块。
- 第 15 阶段：首页看板。
- 第 16 阶段：Excel 导入。
- 第 17 阶段：Excel 导出。
- 第 18 阶段：本地备份恢复。
- 第 19 阶段：异常提醒。
- 第 20 阶段：整体测试和 UI 优化。

## 阶段执行规则

- 用户明确要求执行某一阶段时，只执行该阶段。
- 如果上一阶段缺少必要基础，只补齐最小必要内容。
- 不提前实现后续阶段功能。
- 不擅自重构无关文件。
- 不破坏已经完成的模块。
- 每次改代码前先阅读 `PROJECT_RULES.md`、`ARCHITECTURE.md`、`DATABASE_SCHEMA.md`、`DEVELOPMENT_ROADMAP.md`、`UI_GUIDE.md`。

## 第 1 阶段目标

初始化 Flutter Windows 项目，项目根目录仍必须是当前目录。推荐命令：

```bash
flutter create . --project-name jin_xiao_cun_2026 --platforms=windows
```

如果当前环境不支持 Flutter，应先说明原因，再采用等价方式初始化，但最终项目根目录必须仍然是当前目录。
